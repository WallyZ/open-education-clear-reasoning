# Reviewer Standard

## Purpose

The reviewer standard prevents confident but weak cross-cultural claims. It is not enough for a unit to be interesting. It must be sourced, culturally framed, and safe to teach.

## Review Categories

| Status | Meaning |
| --- | --- |
| `reference_only` | Source can be named, but no excerpt or strong claim should be built from it yet. |
| `needs_source_packet` | A packet is needed before lesson construction. |
| `needs_cultural_review` | Cultural context is not strong enough for confident teaching. |
| `approved_for_original_lesson` | Original lesson may be built without copied source text. |
| `approved_for_excerpt_use` | Narrow source excerpt may be used under documented rights and attribution. |

## Required Reviewer Checks

A reviewer must check:

- source identity and edition are clear
- translation status is clear
- public-domain/license claim is evidenced
- cultural setting is represented accurately enough for the lesson purpose
- Western comparison does not distort the tradition
- lesson names what is transferable and what is not
- adversarial or strategic material includes ethical constraints
- learner-facing claims are appropriately cautious

## Cultural Humility Standard

If the repo lacks enough evidence, mark the unit as `needs_cultural_review`. Do not fill gaps with plausible summaries.

A cautious incomplete note is better than a confident false lesson.

## Reviewer Notes Template

Each reviewed unit should record:

- reviewer name or role
- date
- source packet reviewed
- cultural caveat accepted
- claims approved
- claims rejected or softened
- remaining uncertainties
- allowed lesson use

## Owner Decision Boundary

The owner can decide curriculum priority and sequencing, but source rights and cultural uncertainty should remain evidence-gated. No owner preference should override missing source evidence.
